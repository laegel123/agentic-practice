# 벤치마크 과제 모음

같은 과제를 **(1) 에이전틱/하네스 입히기 전 레거시** 와 **(2) 입힌 후** 에 각각 에이전트에게 시키고,
[agent-scorecard.md](agent-scorecard.md) 에 결과(성공여부·턴수·토큰·시간·권한프롬프트·자가검증·회귀)를 기록한다.

- 에이전트는 **`legacy-ecommerce/` 안에서만** 돌린다. 이 `evaluation/` 폴더(정답지)는 노출하지 않는다.
- 각 과제 프롬프트는 "증상/요구사항"만 준다. 원인 위치는 [answer-key.md](answer-key.md) 에만 있다.
- 한 세션당 한 과제. 새 과제는 깨끗한 상태(또는 새 브랜치)에서 시작한다.

> 채점 팁: 버그 과제는 "원인 정확히 짚었나 / 올바르게 고쳤나 / 회귀 없나 / 테스트 추가했나"로 본다.
> 이해 과제는 정답지의 `file:method` 와 대조한다.

---

## A. 코드 이해 (Comprehension)

- **C1** — 주문 총액(소계·할인·세금·합계)은 어디서, 어떤 순서로 계산되는가? 관련 파일/메서드를 짚어 설명하라.
- **C2** — 주문 도중 결제가 실패하면 어떻게 되는가? 주문 상태와 재고가 어떻게 처리되는지 흐름을 추적하라.
- **C3** — 이 시스템에서 '돈'은 어떤 타입으로 다뤄지고, 반올림은 어디에서 발생하는가? 반올림이 일어나는 지점을 모두 찾아라.
- **C4** — `batch` 모듈은 주문 데이터를 어떻게 읽는가? `ecommerce-service` 와 데이터·스키마 관계를 설명하라.

## B. 버그 수정 (Bug-fix)

- **B1** — 주문할 때마다 합계가 몇 센트씩 어긋난다는 제보. 원인을 찾아 고치고 재발 방지 테스트를 추가하라.
- **B2** — 쿠폰을 적용한 주문이 기대보다 비싸게 청구된다. 세금/할인 계산을 점검해 고쳐라.
- **B3** — 주문 후 재고가 주문 수량의 2배로 줄어든다. 고쳐라.
- **B4** — 환불 누적액이 결제금액을 초과하는 사고가 있었다. 막아라. (관련 에러코드가 이미 정의돼 있다)
- **B5** — '오늘까지 유효'인 쿠폰이 만료됨으로 거부된다. 경계 처리를 고쳐라.
- **B6** — 정산 매출이 실제보다 크다. 취소된 주문이 합산되는 것 같다. 고쳐라.
- **B7** — 상품 목록을 `?page=1` 로 부르면 맨 앞 상품들이 안 나온다. 페이징을 고쳐라.
- **B8** — 어드민 환불 API가 토큰 없이 호출된다. 다른 어드민 API처럼 인증을 적용하라.
- **B9** — 장바구니 요약 합계가 수량을 반영하지 않는다. 고쳐라.
- **B10** — 상품 검색에 SQL 인젝션 취약점이 있다. 안전하게 바꿔라.
- **B11** (고난도) — 자정 무렵 주문이 다음날 매출로 잡힌다. 원인을 찾아 고쳐라.

## C. 기능 추가 (Feature)

- **F1** — 주문 취소 + 환불 엔드포인트를 추가하라. 취소 시 재고를 복원하고 결제 서비스에 환불을 요청한다.
  - 수용 기준: PAID 주문만 취소 가능, 재고 원복, 결제 상태가 환불로 바뀜, 이미 취소된 주문은 거부.
- **F2** — '기프트카드' 결제수단을 추가하라. (잔액 차감 방식)
- **F3** — 상품 리뷰(평점 1~5 + 코멘트) 등록/조회 기능을 추가하라.

## D. 리팩터링 (Refactor)

- **R1** — 금액을 `Money` 값 타입(`common-util`)으로 추출하고, 흩어진 double 연산/중복 반올림을 정리하라. (동작 동일 보장)
- **R2** — `OrderService.placeOrder` 를 책임별(재고/가격/결제/완료처리)로 분해하라.
- **R3** — 어드민의 가격계산 중복을 제거하고 공용 계산을 재사용하라.

## E. 하네스/에이전틱 입히기 (연습의 본체)

- **H1** — 주문 플로우 통합 테스트를 추가하고 `./gradlew test` 로 통과시켜라.
- **H2** — GitHub Actions CI 로 빌드+테스트를 자동화하라.
- **H3** — 루트 `CLAUDE.md` 와 모듈별 컨텍스트/아키텍처 맵을 작성하라.
- **H4** — `.claude/settings.json` 으로 `./gradlew` 류 명령을 허용하고 가드레일 훅을 추가하라.
- **H5** — 정적분석/포매터(Spotless·Checkstyle 등)와 커버리지(Jacoco)를 도입하라.
